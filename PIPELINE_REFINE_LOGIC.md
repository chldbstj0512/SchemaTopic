# Refine 파이프라인 로직: LLM 요청 vs Rule 기반 정제

refine / refine_k / refine_wo 세 모듈별로 **LLM에게 요청한 사항**과 **Rule 기반으로 정제하는 과정**을 표로 정리했다.

---

## 1. refine.py (Auto / Delete 모드)


| 단계         | LLM에게 요청한 사항                                                                                                                                                                                                                             | Rule 기반 정제                                                                                                                                                                                                                                                                                                                                                                           |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Step 1** | • 토픽 단어만 보고 **하나의 의미 기준(criterion)**으로 스키마 설계 • **고수준 스키마 라벨** 제안 (broad category, flat, 단일 라인) • 출력 형식: `CRITERION:` 한 문장 + `SCHEMA:` 아래 `- <label>` 나열                                                                                 | • **flatten_schema_text**: 계층 제거, 최상위 라벨만 유지, `Parent: Child` → Parent만 유지 • **check_schema_step1_flat**: 스키마 형식 검사 • **parse_schema_labels**: SCHEMA 섹션에서 라벨 리스트 추출                                                                                                                                                                                                                 |
| **Step 2** | • 주어진 스키마 + 토픽 단어로 **각 토픽별** (1) 해석 가능성/특이성 평가 (2) 1~2단어 topic_name (3) **keep / delete** 결정 • delete: 불명확·일반·혼합·노이즈·이름 불가 시 • 출력: `topic_id`, `decision`, `topic_name` (keep일 때만) **정확히 N개 JSON 객체**                                    | • **call_llm_until_valid_json**: JSON 파싱/수리·재시도 • **check_and_raise_if_truncated**: 토픽 수 누락 시 TruncationError • **filter_surviving_topics_by_verdict**: decision=delete 제외, keep만 surviving_topics로 (delete된 토픽은 **완전 제거**)                                                                                                                                                          |
| **Step 3** | • **surviving 토픽만** 입력. 각 토픽에 대해 (1) **단어 정리**(topic_name에 맞게 불필요 단어 제거, 6~8단어 유지) (2) **스키마 라벨 할당** • topic_name이 단어와 맞지 않으면 교체, weak면 delete·schema=null • 출력: `topic_id`, `topic_name`, `words`, `schema` **정확히 surviving_topics_n개** | • **postprocess_final_topics**: JSON → 스키마 그룹 구조, 빈/중복/invalid schema → Misc, 단어 3미만 토픽 제외 • **flatten_schema_topics** → refined_topics • Step3에서 누락된 surviving 토픽이 있으면 **Misc로 채워서** refined_topics 보정 • **filter_noise_words**: 2글자 이하 토큰 제거 • **filter_stopwords**: stopword 제거 (tool.py) • **remove_overlapping_words_across_topics**: 2개 이상 토픽에 나온 단어 제거 (min_words_after=1 예외) |

## 2. refine_k.py (K-keep 모드)


| 단계                | LLM에게 요청한 사항                                                                                                                                                                                                                           | Rule 기반 정제                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Step 1**        | • refine과 **동일**: criterion + flat schema 라벨 제안                                                                                                                                                                                        | • **truncate_schema_step1_repetition**: 동일 라인이 3회 이상 연속이면 그 이전까지만 사용 (반복 루프 차단) • **flatten_schema_text**, **check_schema_step1_flat**, **parse_schema_labels** (refine과 동일)                                                                                                                                                                                                                                                                                                                                    |
| **Step 2**        | • refine과 **동일**: 각 토픽 keep/delete + topic_name, 정확히 N개 JSON                                                                                                                                                                           | • **call_llm_until_valid_json**, **check_and_raise_if_truncated** (동일) • **split_keep_and_misc**: keep → surviving_topics, **delete → step2_misc_topics** (topic_name=misc1, misc2, …, schema=Misc). **토픽 수 유지**, delete는 “제거”가 아니라 “Misc로 유지”                                                                                                                                                                                                                                                                  |
| **Step 3**        | • **surviving + misc 전부** 입력. 프롬프트에 “Surviving topics” 블록 + **“Misc topics (word elimination only; schema=Misc)”** 블록 명시 • 각 토픽 (1) 단어 정리 (2) 스키마 할당. **반드시 전체 토픽 수만큼** 객체 출력 (omit 금지) • **refine과 동일** 단어 수 제한(8단어 등), truncation 금지 | • **_ensure_misc_in_schema**: step1 스키마에 Misc 없으면 추가 • **postprocess_final_topics** (refine과 유사, Misc용 topic_name 처리 등) • **merge_step2_misc_into_schema_topics**: step3 결과에 step2 misc 토픽이 빠져 있으면 Misc 그룹에 추가 (misc1, misc2 …) • **_fill_missing_topics_for_keep_mode**: LLM이 누락한 topic_id를 surviving+misc 소스로 복구해 **최종 토픽 수 = 초기 K 유지** • **filter_noise_words**, **filter_stopwords**, **remove_overlapping_words_across_topics** (refine과 동일) • Step1 저장 시 step2에 misc가 있으면 **step1 텍스트 끝에 `- Misc` 추가** 저장 |
| **Truncation 대응** | (Step 3만) 토픽 수가 많을 때 LLM이 잘라낼 수 있음                                                                                                                                                                                                     | • **check_and_raise_if_truncated** 실패 시 **TruncationError** • **_run_step3_chunked_retry**: 토픽을 청크(20개 등)로 나누고, 토픽당 단어 5개로 줄인 프롬프트로 재요청 후 결과 병합                                                                                                                                                                                                                                                                                                                                                                 |

## 3. refine_wo.py (Step Ablation)


| 단계         | LLM에게 요청한 사항                                                                                                       | Rule 기반 정제                                                                                                                                                                                                                                                                                                                                                                                      |
| ---------- | ------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Step 1** | • **skip_step1=False**: refine과 **동일** (criterion + schema 라벨) • **skip_step1=True**: **LLM 호출 없음**                | • skip_step1=True 시 **step1_text = DEFAULT_SCHEMA_TEXT** (CRITERION 한 문장 + SCHEMA: Misc만) • skip_step1=False 시 **flatten_schema_text**, **check_schema_step1_flat**, **parse_schema_labels** (refine과 동일)                                                                                                                                                                                       |
| **Step 2** | • **skip_step2=False**: refine과 **동일** (keep/delete + topic_name, N개 JSON) • **skip_step2=True**: **LLM 호출 없음**    | • skip_step2=True 시 **step2_json = [keep, topic_name=Topic i]** 로 전부 keep 고정, step2_text는 그 JSON 문자열 • skip_step2=False 시 **call_llm_until_valid_json**, **check_and_raise_if_truncated**, **filter_surviving_topics_by_verdict** (refine과 동일)                                                                                                                                                  |
| **Step 3** | • **skip_step3=False**: refine과 **동일** (surviving만 입력, 단어 정리 + 스키마 할당, N개 객체) • **skip_step3=True**: **LLM 호출 없음** | • skip_step3=True 시 surviving_topics를 그대로 **refined_topics**로 만들고, schema=MISC_SCHEMA, words는 기존 words[:20]. **_refined_list_to_schema**로 schema_topics 생성. step3_text는 안내 문구만 • skip_step3=False 시 **postprocess_final_topics**, **flatten_schema_topics**, 누락 토픽 Misc 보정 (refine과 동일) • **filter_noise_words**, **filter_stopwords**, **remove_overlapping_words_across_topics** (refine과 동일) |
| **메타**     | —                                                                                                                  | • schema_meta.json에 **skip_step1, skip_step2, skip_step3** 플래그 저장                                                                                                                                                                                                                                                                                                                               |

## 요약 비교


| 항목               | refine         | refine_k                                              | refine_wo                                         |
| ---------------- | -------------- | ----------------------------------------------------- | ------------------------------------------------- |
| Step 2 delete 의미 | 제거 (step3 미전달) | Misc로 유지 (step3에 전부 전달)                               | refine과 동일 (제거)                                   |
| Step 3 입력        | surviving만     | surviving + misc (전체 K)                               | surviving만 (단, skip_step3 시 LLM 없이 surviving만 사용) |
| Step 1 추가 Rule   | —              | 반복 라인 truncate                                        | skip 시 기본 스키마만                                    |
| Step 2 추가 Rule   | —              | split_keep_and_misc                                   | skip 시 전부 keep                                    |
| Step 3 추가 Rule   | —              | merge_step2_misc, _fill_missing_topics, chunked retry | skip 시 surviving→Misc 스키마만 생성                     |
| 최종 토픽 수          | 삭제된 만큼 감소 가능   | 항상 초기 K                                               | refine과 동일 (skip 조합에 따라 감소 가능)                    |
